Wazuh AI SOC Defense System

CE 4309 Final Project

Intrusion detection and AI-assisted active response for a CPP Bank brute-force login simulation.

Project Goal

  • Detect failed login attacks against a simulated bank app.
  • Extract attacker IP, target user, and failure status.
  • Visualize detections in Wazuh Dashboard / OpenSearch.
  • Use an AI SOC agent to identify brute-force behavior.
  • Block attacker IPs automatically on the defender system.

System Architecture

Attacker Laptop
   -> Windows Defender VM: Flask CPP Bank App
   -> Wazuh Agent reads auth.log
   -> Wazuh Manager decodes + rule 100101 fires
   -> OpenSearch indexes alert
   -> Wazuh Dashboard visualizes alert
   -> AI SOC Agent analyzes alert stream
   -> Windows Defender Firewall blocks attacker on TCP/5000

Attack Log Format

The Flask app writes failed login events into auth.log.

2026-05-07 15:11:43 defense-vm IP=192.168.122.250 | USER=attacker | STATUS=FAILED | REASON=Invalid username or password

Wazuh Detection

Decoder

Extracts srcip, dstuser, and cppbank_status.

Rule 100101

Fires on STATUS=FAILED with level 7.

data.srcip = 192.168.122.250

Dashboard Searches

Failed login detections:

rule.id:100101

AI SOC decisions:

rule.id:100201

AI SOC Agent

  • Runs on the NixOS host, outside the resource-limited Wazuh VM.
  • Polls OpenSearch for rule 100101 alerts.
  • Detects 3+ failed logins in 60 seconds.
  • Indexes AI decisions back to OpenSearch as rule 100201.

Blocking Strategy

Wazuh Manager

iptables DROP rule verifies manager-side response path.

Windows Defender VM

Windows Firewall blocks attacker IP on TCP port 5000.

WINDOWS DEFENDER BLOCKED: 192.168.122.250 on TCP/5000

Admin Safety Guard

The admin host 192.168.122.1 is protected to avoid self-lockout during VM demos.

PROTECTED: 192.168.122.1 matched protected/admin IP list
    
With a separate attacker laptop/IP, this protection does not apply.
