Brute Force Attack Detection & Automated Blocking

Problem Statement

The Challenge

  • Detect brute force SSH attacks in real-time
  • Automatically block attacker IPs
  • Avoid self-lockout (protected admin)
  • Visualize AI decisions in dashboard

Why Traditional Fails

  • Wazuh agent-generated alerts can't trigger active response
  • No AI decision tracking in SIEM
  • No cross-platform blocking (Linux + Windows)

Detection Pipeline

Step 1: Custom Decoder

Extracts srcip from SSH failed login events

decoder name="ssh-failed-login" <parent>sshd</parent> <prematch>Failed password</prematch> <regex offset="after_parent">(\S+) from (\S+) port</regex> <order>src_ip</order>

Step 2: Rule 100101

Triggers on 3+ failed logins, extracts IP

<rule id="100101" level="10"> <decoded_as>ssh-failed-login</decoded_as> <description>SSH Brute Force Attack</description> <field name="srcip">^\\d+</field> </rule>

AI SOC Agent

Detection Logic

  • Polls OpenSearch every 10 seconds
  • Counts failed logins per IP
  • Threshold: 3+ attempts in 60s
  • Protects admin (192.168.122.1)

Decision Indexing

Writes AI decisions back as rule.id:100201

  • Block action recorded
  • Visible in dashboard
  • Audit trail for SOC

Active Response

Linux (iptables)

iptables -A INPUT -s {attacker_ip} -j DROP

Applied on Wazuh Manager

Windows Defender Firewall

netsh advfirewall firewall add rule name="AI Block {ip}" dir=in action=block remoteip={ip} protocol=TCP localport=5000

Blocks on Windows Defender VM
